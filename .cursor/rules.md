# Cursor Project Rules

## Project Information

This project is developed using AI-assisted development practices.

## Technology Stack

- Node.js LTS
- JavaScript
- Git
- GitHub
- Cursor IDE

## Coding Standards

- Use clean and readable code.
- Follow JavaScript ES6+ standards.
- Use meaningful names for variables and functions.
- Keep functions small and focused.
- Add comments when logic is complex.

## File Organization

- Keep source code inside the `src` folder.
- Keep documentation files in the project root.
- Avoid unnecessary files.

## Git Guidelines

All commits should follow Conventional Commits format.

Examples:

- `feat:` for new features
- `fix:` for bug fixes
- `docs:` for documentation changes
- `refactor:` for code improvements
- `test:` for test-related changes
- `chore:` for configuration changes

## Cursor AI Usage

Cursor AI should be used to:

- Review code quality
- Suggest improvements
- Help debug issues
- Improve documentation
- Assist with development tasks

AI-generated code should always be reviewed before use.
